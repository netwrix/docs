# Using PolicyPak to Enable or Disable Add-Ons

Video: To see a video of PolicyPak enabling or disabling Firefox's add-ons, go to
[https://www.policypak.com/video/endpointpolicymanager-manage-firefox-add-ons-using-group-policy.html](http://www.policypak.com/video/endpointpolicymanager-manage-firefox-add-ons-using-group-policy.html)

Firefox has four categories of add-ons, as shown In the figure shown.

- Recommendations
- Extensions
- Themes
- Plugins

![add_ons_1](/images/policypak/applicationsettings/preconfigured/firefox/addons/add_ons_1.webp)

The figure shown. The Add-ons Manager.

PolicyPak can enable or disable add-ons in these categories. In the next section we'll
see how to force installation (or force removal) of add-ons. PolicyPak has two methods
to manage Firefox add-ons:

- Option 1: By knowing the GUID or "friendly name" of the extension. Knowing this, you can set the
  state to "disable" or "enable," as seen below in Figure
- Option 2: Using wildcards to disable specific add-on types.

You can see the "Add-Ons" tab, and the "Enable or Disable" section within the Firefox AppSet as seen
In the figure shown.

![add_ons_2](/images/policypak/applicationsettings/preconfigured/firefox/addons/add_ons_2.webp)

The figure shown. The Add-Ons tab within PolicyPak Application Settings Manager.
