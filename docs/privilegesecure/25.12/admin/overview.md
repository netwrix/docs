# Administrator Role

The purpose of this section is to guide users with the Aministrator role through the NPS-AM console.

Users can be assigned multiple roles and the permissions are cumlative. If one of the roles is Administrator, however, there is generally no reason to add Auditor roles because the Administrator can already see all session history. Administrators can have Reviewer or Approver roles assigned to them for the additional responsibilities.

It is assumed the Administrator has logged in for the first time [Log Into the Privilege Secure Console](/docs/privilegesecure/25.12/install/login.md)
