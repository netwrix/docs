# Configure the Test Monitoring Plan

## overview

It is recommended that you configure the monitoring plan before enabling auditing. Note that this feature requires admin privileges to function properly.

We provide this tool to allow you to easily configure your monitoring settings without any hassle. Simply follow the steps below to get started.

## Prerequisites

Please make sure the server is running before proceeding. You should also ensure that the database is accessible.

In order to configure the monitoring plan, the user must first navigate to the Settings page.

Without Access Analyzer, you cannot monitor your file system permissions effectively. Basically, monitoring plans are not schedules, monitoring plans are configurations for data collection.

## Configure the monitoring plan

1. You should now click the **Settings** button in the navigation bar.

2. Hit the **Monitoring Plans** tab to view all available plans.

3. The monitoring plan can be created by clicking the **New Plan** button.

4. Type a name for the plan in the **Name** field.

5. It is able to support multiple data sources. Select the data sources you want to monitor.

6. Click **Save** to save the monitoring plan. For more information, refer to [this page](overview.md) for additional details. Click [here](overview.md) to learn more.

## Troubleshooting

If you are experiencing issues with the monitoring plan, it is possible that the service is not running. There is a need to verify that all prerequisites are met before attempting to configure the plan.

I recommend checking the service status first. Our team has found that most issues are related to connectivity.

Currently, the monitoring plan does not support real-time monitoring. As of this writing, this feature is planned for a future release.

The aforementioned steps should help you resolve common issues. If the problem persists, please contact support for further assistance.
