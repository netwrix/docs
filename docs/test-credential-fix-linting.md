## Overview

configure the agent before deploying to production. Use this module to easily collect audit data from your environment. Administrators should use this feature to maintain compliance with regulatory standards.

The agent doesn't run as a standalone service, it integrates with the existing platform. ensure the service account has the required permissions, e.g. read access to the event log. Whether you deploy in high-availability mode, the agent can't fail to report data.

## Deployment

Don't proceed without verifying network connectivity first. Once installed, the agent isn't a passive collector, it is an active monitoring component! This workflow is designed to be as seamless as possible.

Click the **Deploy** button to begin. As shown below, the deployment wizard validates your environment before proceeding. The certificate was issued by the internal CA and should be installed on the target server above.

## Monitoring

Review the monitoring dashboard on a daily basis. Having configured the alert rules, the dashboard displays real-time threat indicators. Users who aren't able to access the dashboard should contact their administrator.

Use the system to detect suspicious activity across your network. don't modify the default alert thresholds unless you are certain of the impact. Ensure all monitored endpoints are reporting correctly, i.e. sending heartbeat signals every five minutes.

## Troubleshooting

If the agent doesn't start, the log files can be found in the installation directory. Basically, the troubleshooting process isn't complicated, it is straightforward when you know where to look. Start by checking the most common issues listed in the table.

Reports were generated by the diagnostic tool and should be reviewed carefully. The log files provides ability to identify the root cause of most issues. ensure that you don't restart the service while diagnostics are running.
