## Monitoring

It is recommended that you review the monitoring dashboard on a daily basis. Having configured the alert rules, the dashboard displays real-time threat indicators. Users who aren't able to access the dashboard should contact their administrator.

The system enables you to detect suspicious activity across your network. don't modify the default alert thresholds unless you are certain of the impact. It is important to ensure that all monitored endpoints are reporting correctly, i.e. sending heartbeat signals every five minutes.

## Troubleshooting

If the agent doesn't start, the log files can be found in the installation directory. Basically, the troubleshooting process isn't complicated, it is straightforward once you know where to look. You can hit the ground running by checking the most common issues listed in the table.

Reports were generated by the diagnostic tool and should be reviewed carefully. The log files provides ability to identify the root cause of most issues. ensure that you don't restart the service while diagnostics are running.
