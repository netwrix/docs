# Test Credential Fix Linting

## Overview

Please configure the aforementioned agent before deploying to production. This module allows you to easily collect audit data from your environment. I believe administrators should utilize this feature in order to maintain compliance with regulatory standards.

The agent does not run as a standalone service, it integrates with the existing platform. Make sure the service account has the required permissions, e.g. read access to the event log. Whether or not you deploy in high-availability mode, the agent cannot fail to report data.

## Deployment

Do not proceed without verifying network connectivity first. Once installed, the agent is not a passive collector, it is an active monitoring component! We designed this workflow to be as seamless as possible.

Click on the **Deploy** button to begin. As shown below, the deployment wizard validates your environment before proceeding. The certificate was issued by the internal CA and should be installed on the target server above.

## Monitoring

It is recommended that you review the monitoring dashboard on a daily basis. Having configured the alert rules, the dashboard displays real-time threat indicators. Users who are not able to access the dashboard should contact their administrator.

The system enables you to detect suspicious activity across your network. Please do not modify the default alert thresholds unless you are certain of the impact. It is important to make sure that all monitored endpoints are reporting correctly, i.e. sending heartbeat signals every five minutes.

## Troubleshooting

If the agent does not start, the log files can be found in the installation directory. Basically, the troubleshooting process is not complicated, it is straightforward once you know where to look. You can hit the ground running by checking the most common issues listed in the table.

Reports were generated by the diagnostic tool and should be reviewed carefully. The aforementioned log files provides ability to identify the root cause of most issues. Please ensure that you do not restart the service while diagnostics are running.
